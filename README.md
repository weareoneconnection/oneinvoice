# OneInvoice AI

Malaysia F&B e-Invoice Compliance Platform MVP.

This code package is a runnable full-stack Next.js MVP for restaurant e-Invoice workflows:

- Dashboard for sales, SST, consolidated pool and pending issues
- POS CSV import
- Receipt list with customer e-Invoice request links
- Customer e-Invoice request form
- Individual e-Invoice mock submission
- Monthly consolidated e-Invoice batch generation
- MyInvois sandbox/mock gateway center
- AI Accountant deterministic assistant
- Local JSON database for quick demo

## Quick start

```bash
npm install
npm run seed
npm run dev
```

Open:

```text
http://localhost:3000
```

## Demo flow

1. Run `npm run seed` to generate demo restaurant receipts.
2. Open `/dashboard` to see compliance status.
3. Open `/receipts`, click a receipt's customer link, and submit a buyer e-Invoice request.
4. Open `/requests` to view buyer request status.
5. Open `/consolidated`, select the current month, and generate a consolidated e-Invoice batch.

## CSV import format

The importer accepts flexible field names. Recommended columns:

```csv
receiptNo,outlet,channel,date,subtotal,serviceCharge,sst,discount,rounding,total
R001,KLCC Outlet,dine_in,2026-06-01T12:00:00.000Z,100,0,6,0,0,106
R002,Main Outlet,takeaway,2026-06-01T13:00:00.000Z,50,0,3,0,0,53
```

Also supported aliases:

- `receipt_no`, `bill_no`, `invoice_no`, `order_no`
- `branch`, `location`
- `net_total`, `amount_excluding_tax`, `sales`
- `service_charge`, `svc_charge`
- `tax`, `tax_amt`
- `grand_total`, `payable_amount`

## Production integration points

### 1. MyInvois gateway

Replace mock functions in:

```text
lib/myinvois.ts
```

with official MyInvois API calls:

- Login as taxpayer system
- Validate Taxpayer TIN
- Submit documents
- Get submission status
- Get document
- Cancel / reject document
- Search documents

### 2. OneAI integration

Replace deterministic helper logic in:

```text
lib/ai.ts
app/api/ai/chat/route.ts
```

with OneAI `/v1/chat/completions` or `/v1/generate` calls for:

- POS CSV field mapping
- MyInvois error explanation
- Monthly accountant pack generation
- Multilingual customer support
- Compliance checklist generation

### 3. Database

For production, replace `data/db.json` with PostgreSQL + Prisma.
Recommended tables:

- companies
- outlets
- users
- receipts
- receipt_items
- customer_requests
- einvoices
- consolidated_batches
- api_logs
- ai_reports

## Notes

This is an MVP framework, not a certified tax product. Before production use, validate against LHDN / IRBM e-Invoice requirements, MyInvois API specifications, accountant review, privacy requirements, and payment/subscription controls.
